# HaloHex PATH AI

**AI-Powered Personalized Learning Path Generator**
Theme: Data-Driven Adaptive Learning System

HaloHex PATH AI turns a career goal, skill level, and available study time
into a full personalized roadmap: a week-by-week plan, curated resources,
mini projects, a capstone project, and an estimated completion timeline —
generated by Google's Gemini AI.

---

## ✨ Features

- Modern glassmorphism SaaS-style UI (blue/purple gradient, hex-path motif)
- Simple form: name, career goal, skill level, daily study hours, duration
- AI-generated roadmap via Gemini (with a built-in offline demo fallback,
  so the app works even without an API key)
- Every generated roadmap is saved to a local SQLite database
- Clean FastAPI backend with SQLAlchemy models

## 🖥 Frontend Pages

The frontend is now a full multi-page app (still plain HTML/CSS/JS, no build step):

| Page | File | Notes |
|------|------|-------|
| Landing | `landing.html` | Marketing homepage. `index.html` redirects here. |
| Login | `login.html` | Placeholder auth — see note below. |
| Register | `register.html` | Placeholder auth — see note below. |
| Dashboard | `dashboard.html` | Overview stats + quick actions. |
| Generate Learning Path | `generate.html` | The original AI roadmap generator, wired to the FastAPI backend. |
| Roadmap Progress | `progress.html` | Checkbox tracker over a sample roadmap. |
| Profile | `profile.html` | Edit learning preferences & account details. |

All logged-in pages (Dashboard, Generate, Progress, Profile) share a sidebar
and a common `app.js`, which currently uses a hardcoded `CURRENT_USER` object
since there's no auth backend yet.

### ⚠️ Important — no real auth/persistence yet

Login, Register, Progress, and Profile currently work with **placeholder,
in-memory logic only** — nothing is saved to a database and there is no real
authentication. Each file has `TODO`-style comments showing exactly what to
replace. In short, to make this production-ready you'd want to:

1. Add a separate **accounts** table + `POST /register`, `POST /login`,
   `GET /me` endpoints to the backend. Don't reuse the existing `users`
   table in `models.py` — that one stores generated roadmaps, not login
   credentials.
2. Have login set a real session (HttpOnly cookie or token), and have
   `app.js` fetch the logged-in user from `GET /me` instead of using the
   hardcoded `CURRENT_USER` object.
3. Add a way to persist roadmap-progress checkboxes (e.g. a
   `completed_topics` field/table keyed by user + roadmap id), and have
   `progress.html` load/save against it instead of `SAMPLE_ROADMAP`.

### 1. Backend setup

```bash
cd backend
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

**(Optional but recommended) Add your Gemini API key.**
Get a free key at https://aistudio.google.com/apikey, then set it as an
environment variable before starting the server:

```bash
# macOS / Linux
export GEMINI_API_KEY="your-key-here"

# Windows (PowerShell)
setx GEMINI_API_KEY "your-key-here"
```

> If you skip this step, the app still works — it will automatically use a
> built-in demo roadmap generator so you can test and demo the full flow.

**Run the server:**

```bash
uvicorn main:app --reload
```

The API will be live at `http://127.0.0.1:8000` (interactive docs at
`http://127.0.0.1:8000/docs`).

### 2. Frontend setup

No build step needed — it's plain HTML/CSS/JS.

Just open `frontend/landing.html` directly in your browser
(double-click it, or right-click → "Open with" your browser).
`frontend/index.html` will also redirect there automatically.

The frontend is pre-configured to call the backend at
`http://127.0.0.1:8000` (see the `BACKEND_URL` constant at the top of
`script.js` — change it if your backend runs elsewhere).

## 🧠 How it works

1. The user fills in their name, career goal, skill level, daily study
   hours, and target duration in months.
2. The frontend sends this to `POST /generate-roadmap`.
3. The backend builds a detailed prompt and calls the Gemini API, asking
   for a strict JSON response containing the roadmap, resources,
   projects, capstone project, estimated completion, and a motivational
   message.
4. The result is saved to the `users` table in `database.db` and
   returned to the frontend.
5. The frontend renders everything into styled cards.

## 🔌 API Reference

### `POST /generate-roadmap`

**Request body:**

```json
{
  "name": "Priya",
  "goal": "Machine Learning Engineer",
  "skill_level": "Beginner",
  "daily_hours": 2,
  "duration": 6
}
```

**Response:** the saved record, including the full `roadmap` object
(week-by-week topics, resources, projects, capstone, completion
estimate, motivation message).

### `GET /history`

Returns every roadmap ever generated, most recent first — useful for a
judges' demo of "look, it's actually persisting data."
